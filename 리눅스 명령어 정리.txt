poweroff : 시스템 종료
shutdown : 시스템 종료
    ㄴ> -r 시스템 재시작
    ㄴ> -P 시스템 강제 종료 ( 소문자 안됨 )
    ㄴ> -h 시스템 종료
    ㄴ> -c 예약된 셧다운 명령어 취소
    ㄴ> -k 명령 수행하지 않고, 경고 메시지만 전달
    ㄴ> +m m분 후에 시스템 셧다운 
    ㄴ> 숫자 : (shutdown 5) 5분뒤에 시스템 종료
halt : 시스템 종료 로그를 남기고 종료 or 재부팅
    ㄴ> -d wtmp파일에 로그 남기지 않음
    ㄴ> -f 강제 종료
    ㄴ> -n 종료할 때 싱크 X
    ㄴ> -w 종료 할 때 로그만 남김
    ㄴ> -p 강제 종료
init : 몇 기능을 함
    ㄴ> 0 시스템 종료 (halt)
    ㄴ> 1 Single 모드로 쉘로 바로 부팅
    ㄴ> 2 NFS 기능을 사용하지 않는 멀티 유저 모드
    ㄴ> 3 모든 기능을 포함한 멀티유저 모드, 콘솔 부팅
    ㄴ> 4 사용하지 않음
    ㄴ> 5 X-윈도우로 부팅 및 멀티유저 모드
    ㄴ> 6 시스템 재부팅
logout, exit : 로그아웃 또는 exit은 터미널 나가기 가능.
cd : (change directory) 현재 실행중인 터미널의 디렉토리 변경
ls : 디렉토리에 있는 내용 출력
    ㄴ> -a (all) 숨겨진 파일이나 디렉토리도 보여줌.
    ㄴ> -l (long) 자세한 내용 출력 (퍼미션, 포함된 파일 수, 소유자, 그룹, 파일크기 등)
    ㄴ> -S (size) 파일 크기순으로 정렬하여 출력
    ㄴ> -r (reverse) (기본은 알파벳 순서) 거꾸로 출력
    ㄴ> -R (recursive) 하위 디렉토리 까지 출력
    ㄴ> -h (human) K, M, G 단위를 사용하여 파일크기를 사람이 보기 좋게 표시
    ㄴ> -lu 접근시간 출력
    ㄴ> -lc 변경 시간 출력
history : 디렉토리 내 변경점 출력
    ㄴ> -c 히스토리 삭제
    ㄴ> -d offset : offset 위치에 있는 history 명령어 삭제
    ㄴ> -a history list에 있는 내용들을 파일에 추가
    ㄴ> -r history list에 히스토리 파일을 읽고 내용 추가
    ㄴ> -w 현재 history를 히스토리 파일에 씀.
cat : 파일 또는 텍스트 파일을 화면에 출력함.
    ㄴ> cat a.txt b.txt 라고 입력 시 a 파일 뒤에 b 파일을 연결하여 출력
-gedit 에디터
    ㄴ> gedit은 X 윈도우에서 제공하는 편리한 에디터로, 윈도우의 메모장과 비슷
    ㄴ> [ gedit 파일명 ] 으로 해당 파일 편집 가능
    
-vi 에디터
    ( ESC 후 )
    ㄴ> :q - vi 에디터 종료
    ㄴ> IA ( Insert, Append ) : 입력모드 입력
    ㄴ> :wq - 저장 후 종료
    ㄴ> :q! - 변경 점 무시하고 종료
    - 명령 모드에서 입력 모드로의 전환 키
        ㄴ> i (현재 커서 위치부터 입력)
        ㄴ> a (현재 커서 다음 칸부터 입력)
        ㄴ> o (현재 커서의 다음 행에 입력)
        ㄴ> s (현재 커서의 한 글자를 지우고 입력)
        ㄴ> I (현재 커서 행의 맨 앞에서부터 입력)
        ㄴ> A (현재 커서 행의 맨 마지막에서부터 입력)
        ㄴ> O (현재 커서 앞행에 입력)
        ㄴ> S (현재 커서의 행을 지우고 입력)
    - 명령 모드에서 커서를 이동하기 위한 전환 키
        ㄴ> h (커서를 왼쪽으로 한 칸 이동)
        ㄴ> j (커서를 아래로 한 칸 이동)
        ㄴ> l (커서를 오른쪽으로 한 칸 이동)
        ㄴ> k (커서를 한칸 위로 이동)
        ㄴ> CTRL+F (다음 화면으로 이동)
        ㄴ> CTRL+B (이전 화면으로 이동)
        ㄴ> ^ (현재 행의 처음으로 이동)
        ㄴ> $ (현재 행의 마지막으로 이동)
        ㄴ> gg (첫 행으로 이동)
        ㄴ> G (마지막 행으로 이동)
        ㄴ> 숫자+G ( 숫자 행으로 이동)
        ㄴ> :숫자 ( 숫자의 행으로 이동)
    - 명령 모드에서의 삭제, 복사, 붙여넣기 관련 키
        ㄴ> x (현재 커서가 위치한 글자 삭제)
        ㄴ> X (현재 커서의 앞 글자 삭제)
        ㄴ> dd (현재 커서의 행 삭제)
        ㄴ> 숫자+dd (현재 커서부터 숫자만큼의 행 삭제)
        ㄴ> yy (현재 커서가 있는 행 복사)
        ㄴ> 숫자+yy ( 현재 커서부터 숫자만큼의 행 복사 )
        ㄴ> p ( 복사한 내용을 현재 행 뒤에 붙여넣기 )
        ㄴ> P(대문자임이거) ( 복사한 내용을 현재 행 앞에 붙여넣기 )
    - 명령 모드에서 문자열을 찾는 키
        ㄴ> /문자열 ( 해당 문자열을 찾음 ( 현재 커서 이후로 ))
        ㄴ> n (찾은 문자 중에서 다음 문자로 이동)

    ㄴ> -f - 강제로 삭제
    ㄴ> -r - 디렉토리 내 모든 내용 삭제

    ㄴ> -o 항목 - 마운트 할 때 추가적인 설정을 적용할 때 사용하는 옵션

dpkg, apt-get: 우분투에서 패키지를 설치할 때 가장 많이 사용되는 명령어
    ㄴ> apt-get은 dpkg의 확장형
    - dpkg 옵션
        ㄴ> -i ( install ) 패키지 설치
        ㄴ> -r ( remove ) 패키지 제거
        ㄴ> -P ( purge ) 패키지와 설정 파일 모두 삭제
        ㄴ> -L 패키지 정보 출력
        ㄴ> --info 패키지파일명.deb : 아직 설치되지 않은 deb 파일 조회
    - apt-get 옵션
        ㄴ> install
        ㄴ> update
        ㄴ> remove
        ㄴ> purge
        ㄴ> autoremove ( 사용하지 않는 패키지 모두 삭제 )
        ㄴ> clean/autoclean ( 설치할 때 다운로드한 파일과 과거 파일 제거 )

apt-cache : 패키지 설치 전에 패키지에 대한 정보나 의존성 문제 미리 확인
    ㄴ> apt-cache show 파일명 : 파일명 패키지 정보 출력
    ㄴ> apt-cache depends 파일명 : 패키지 의존성 정보 출력
    ㄴ> apt-cache rdepends 파일명 : 패키지에 의존하는 다른 패키지 목록 출력
